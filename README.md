# coding-exam : Programming Quiz System (LTI Tool)

このプロジェクトは、プログラミング課題の解答として学習者から提出されたソースコードをAIで解析し、学習者ごとにパーソナライズされた理解度チェック（自分が提出したコードの意味をちゃんと理解しているか）のクイズを自動生成する学習支援ツールです。

LMSからLTI1.1での呼び出しによって利用します。動作検証は Ubuntu 22.04, Moodle 4.5 で行っています。

---

## 学習者向け：クイズ解答画面

ソースコードの特定の行（重要箇所）について、その役割や意味を正しく理解しているかを問うインターフェースです。2カラムレイアウトになっており、左側にソースコード、右側に問題文と選択肢が表示され、コードを参照しながら解答することが可能です。

### 主な機能

* **動的問題生成**: 重要度の高い10行を毎回ランダムに抽出し、出題します。
* **シャッフル機能**: 正解の選択肢と誤答（distractor）を混ぜ合わせ、実行のたびに異なる選択肢順で提示します。
* **即時フィードバック**: 「練習モード」が有効な場合、提出後すぐに各行の正誤判定と正解を確認できます。

### 利用フロー

1. **解答**: プルダウンメニューから各行の適切な説明を選択します。
2. **提出**: [解答を提出する] ボタンをクリックすると、解答とスコアが保存されます。
3. **復習**: 練習モード時は、正解と自分の解答を比較した表が即時に表示されます。

## 教員向け：管理パネル

LMSのリソース（このシステムへのLTIリンク）ごとに、クイズの出題設定をカスタマイズし、学習者の成績を管理できます。

### 主な機能

* **クイズ設定**:
  * **ExamID の紐付け**: `resource_link_id`（LMSのリソース単位）ごとに、出題対象となる試験IDを設定・変更できます。
  * **練習モードの制御**: 学習者が提出直後に正解や詳細解説を確認できる「練習モード」の ON/OFF を制御可能です。


* **成績管理**:
  * **成績データのフィルタリング:** 全成績データの中から、現在表示しているリソースに紐づく結果のみを抽出して表示します。
  * **リアルタイム集計**: 受験した学習者の最高スコアを自動的に算出して一覧表示します。
  * **CSVダウンロード**: 成績一覧を CSV 形式（UTF-8 BOM付き）でダウンロードし、外部のシステム（Moodle等）へのインポート等に利用できます。



### 操作フロー

1. **設定**: 試験IDを入力し、練習モードの有無を選択して [保存] をクリックします。
2. **確認**: 画面下部のデータテーブルで、学習者ごとの最高得点を確認します。
3. **出力**: [成績データをダウンロード] ボタンから、CSVファイルを保存します。

## 言語の切り替え (日本語 / 英語)

本システムは日本語か英語での表示が可能です。

LMS（Moodleなど）側の現在の言語設定に従って、本システムを呼び出した時の表示言語も変わります。

LMSから送信される LTI リクエスト内の `launch_presentation_locale` パラメータを読み取っています。
  * `ja` を含む場合：日本語表示
  * それ以外：英語表示

## セットアップ

### 1. 依存ライブラリのインストール

```bash
pip3 install streamlit fastapi uvicorn requests pandas google-genai openpyxl

```

### 2. クイズデータの生成

あらかじめ学習者の提出コードをまとめた `extracted_code_with_userid.xlsx` を用意してください。以下のコマンドによってAIによる問題生成を実行できます。

```bash
cd quizdata; python3 xlsx2quizdata.py

```

#### `extracted_code_with_userid.xlsx`の構造

クイズ生成の元となる Excel ファイルは、以下の列名を持つ必要があります。このデータを用いて、`xlsx2quizdata.py` が各学習者専用のクイズを自動生成します。

| 列名 | 説明 |
| --- | --- |
| `username` | 学習者の識別子（ユーザのIDナンバーなど）。LTIから渡されるユーザー名と一致させる必要があります |
| `examid` | 課題の識別番号（例: `1`, `2`）。教員パネルで指定する ID と連動します |
| `extractedcode` | 出題対象となるソースコード（学習者が自分で提出したもの） |
| `filename` | （任意）提出時のファイル名。管理用に使用されます |

なお、quizdata/extract_code_and_save.ipynb を用いると、Moodleの課題に提出された各学習者の提出ファイルからプログラムのコードを抽出して、extracted_code_with_userid.xlsx に必要な構造を持つファイルを作成することができます。

### 3. サーバーの起動

`main.py` を実行すると、FastAPI（ポート8803）とStreamlit（ポート8802）が同時に起動します。
（8802 と 8803 のポートが、外部からアクセスできるようにしておく必要があります。）

```bash
python3 main.py

```

## LMSとの連携のための設定

LMSとの連携を安全に行うため、`main.py` 内の以下の変数を適切に設定してください。

* **`LTI_CONSUMER_KEY`**:
LMS側で設定するLTI呼び出し（Moodleでは「外部ツール」）の「コンシューマーキー」と一致させる必要があります。任意の文字列を指定してください。
* **`ALLOWED_LMS_HOST`**:
呼び出す元のLMSのドメインを指定します。
  * 特定のホストに限定する場合（例）: `ALLOWED_LMS_HOST = "moodle.your.domain"`
  * 構築中などで制限を設けない場合: `ALLOWED_LMS_HOST = "ALL"`
* OAuth 署名検証は行っておらず、Consumer Key 照合のみを行う簡易実装です。

## ファイル構成

| ファイル名 | 役割 |
| --- | --- |
| `main.py` | **LTI認証サーバー**: LTIリクエストを検証し、ワンタイムトークンを発行 |
| `app.py` | **メインアプリ**: 認証チェック後、ユーザーのロールに応じたパネルを表示 |
| `auth.py` | **認証ブリッジ**: FastAPIとStreamlit間のトークン検証を仲介 |
| `components/quiz.py` | **学習者用画面**: クイズの解答、採点、練習モード時の結果表示 |
| `components/instructor.py` | **教員用パネル**: 試験ID設定、練習モード切替、成績CSV出力 |
| `components/admin.py` | **管理者画面**: 全ユーザーの解答状況とソースコードのハイライト表示 |
| `database.py` | **DB管理**: SQLiteへの読み書きを担当 |
| `utils.py` | **多言語・UI管理**: 言語リソース定義とコピペ防止CSSの注入 |
| `quizdata/xlsx2quizdata.py` | **生成AIでの問題データの生成**: Gemini を使用してコードを解析し問題を作成 |
| `quizdata/extract_code_and_save.ipynb` | Moodleの課題提出ファイルからプログラムのコードを抽出してデータファイルを作る |
